Title...........: VB5THREADS: Threading Demo
Author..........: JKWild1
Release Notes...: This file creates real threading in your VB apps by using the win API and AddressOf. All code is commented to some extent. You can *prove* the existence of multiple threads in a number of ways... one is using a process viewer such as a wintop (do a software search for it or as power toys). The second is in the application... normally Do 'code loop without a doevents would cause your program to become unresponsive, however if that code is placed in a separate thread from your main thread using the included code mod and class, you program will not become unresponsive... therefore you can perform intensive calculations, etc while doing other things... even better though is that you set different priorities for different threads. -JKWild1
